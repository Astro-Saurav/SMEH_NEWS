# Security Policy

## Supported Versions

We provide security updates and patches for the following versions:

| Version | Supported |
| ------- | --------- |
| 1.x.x   | Yes       |
| < 1.0.0 | No        |

## Reporting a Vulnerability

We take the security of SMEH_NEWS seriously. If you find a security vulnerability, please do **not** open a public issue. Instead, report it responsibly through one of the following channels:

* **GitHub Security Advisory**: Submit a private vulnerability report via the "Security" tab of this repository.
* **Direct Contact**: Contact the maintainer [Saurav](https://github.com/Astro-Saurav) directly via profile contacts.

### What to Include in a Report
Please include as much detail as possible to help us reproduce and fix the issue quickly:
* A description of the vulnerability.
* A proof of concept (PoC) or step-by-step instructions to reproduce it.
* The potential impact of the vulnerability.

### Our Response Process
1. We will acknowledge receipt of your report within 48 hours.
2. We will investigate the issue and determine a remediation plan.
3. We will keep you updated on our progress as we work towards a fix.
4. Once a fix is ready, we will release a new version and publicly credit you for the discovery (if desired).
